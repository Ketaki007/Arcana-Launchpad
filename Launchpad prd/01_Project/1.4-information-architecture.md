# Information Architecture

## Why This Document Exists

This document defines the structural hierarchy and shared vocabulary of Arcana Launchpad.

## Product Hierarchy

Workspace
└── Views
    └── Pages
        └── Widgets
            └── Neo Components

## Workspace

A persistent environment configured around a user's responsibilities and workflows.

## View

A high-level workflow or area of focus within a workspace.

## Page

A saved layout supporting a specific analytical activity.

## Widget

A focused analytical tool such as Watchlist, Chart, News, Financial Analysis, or Relative Valuation.

## Shared Context

Widgets participate in a shared analytical context, allowing selections and state to propagate across the workspace.

## Investigation

An ongoing analytical activity that emerges from the combination of workspace, view, page, widget state, and user actions.

## Intelligent Layers

Cross-workspace capabilities that observe context and assist users without replacing decision-making.

## Interaction Enhancements

Capabilities that improve existing workflows without changing the product structure.

## Navigation Model

Workspace → View → Page → Widget

Navigation should preserve analytical context wherever possible.

## Design Implications

Every new feature should fit naturally within this hierarchy and avoid introducing unnecessary product concepts.
