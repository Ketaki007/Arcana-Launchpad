# Design Language

## Purpose

The Neo Design Language defines the visual philosophy for AI-generated enterprise analytical products.

It prioritizes analytical efficiency over visual comfort.

The interface should help users process large amounts of information quickly and accurately during extended working sessions.

Every design decision should increase clarity, reduce friction and support analysis.

---

# Design Personality

Neo should feel:

- Professional
- Analytical
- Calm
- Precise
- Dense
- Efficient
- Trustworthy
- Focused
- Minimal
- Fast

The interface should resemble a professional workstation rather than a modern consumer application.

---

# Core Philosophy

## Data First

Data is the primary interface.

Users spend most of their time analysing information rather than interacting with controls.

Give maximum space to data.

Minimise space used by chrome.

---

## Information Density

Neo intentionally favours high information density.

Whitespace exists only where it improves readability.

Avoid generous spacing that reduces visible information.

The interface should comfortably display large datasets without feeling cluttered.

---

## Workspace Before Dashboard

Neo generates analytical workspaces—not marketing dashboards.

Widgets work together to support continuous analysis.

The workspace should feel persistent, configurable and purpose-built.

---

## Tables Before Cards

Tables are the default representation for analytical information.

Use cards only when presenting:

- KPIs
- summaries
- alerts
- single metrics

Whenever users compare, scan, rank or investigate data, prefer tables.

---

## Charts Support Analysis

Charts are investigative tools.

They support decision making.

Avoid decorative visualizations.

Prefer:

- thin strokes
- visible gridlines
- multiple comparable series
- dense axes
- compact legends

Charts should maximize data area.

---

## Panels, Not Cards

Widgets are analytical panels.

Panels should:

- have visible borders
- have minimal radius
- have no shadows
- contain integrated headers
- maximise usable content area

Avoid floating card aesthetics.

---

## Compact Controls

Controls should consume the minimum practical space.

Optimise for:

- mouse
- keyboard
- power users

Buttons, dropdowns, tabs and inputs should never dominate the interface.

---

## Progressive Disclosure

Expose complexity gradually.

Show:

- summary first
- details on demand
- advanced actions contextually

The interface should feel powerful without appearing overwhelming.

---

# Visual Hierarchy

Hierarchy should come from:

- typography
- alignment
- spacing
- grouping

—not through decoration.

Avoid using:

- large typography
- oversized icons
- excessive color
- unnecessary emphasis

---

# Color Philosophy

Color communicates meaning.

Never use color purely for decoration.

Typical semantic usage:

Green
Positive

Red
Negative

Blue
Selection

Yellow
Warning

Gray
Neutral

---

# Motion

Motion should be subtle.

Avoid animations that distract from analytical work.

Prefer:

- instant feedback
- quick transitions
- stable layouts

---

# Anti-Patterns

Avoid generating interfaces that resemble:

- Consumer SaaS dashboards
- Marketing websites
- Mobile-first layouts
- Touch-first interfaces
- Glassmorphism
- Large floating cards
- Oversized border radius
- Soft shadows
- Decorative gradients
- Excessive whitespace
- Large hero sections
- Oversized typography
- Empty layouts

---

# AI Interpretation

When generating interfaces:

- Prioritize information density.
- Prefer panels over cards.
- Prefer tables over summaries.
- Keep controls compact.
- Maximize usable data area.
- Use color semantically.
- Generate interfaces for professional users working for long periods.
- Assume keyboard and mouse are the primary interaction methods.
- Every pixel should contribute to analysis.
